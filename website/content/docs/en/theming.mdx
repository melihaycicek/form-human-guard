The `SubmitGuard` overlay is themed with a **preset + override** model:

```tsx
<SubmitGuard theme="glass" />
<SubmitGuard theme="terminal" themeOverrides={{ accent: "#f59e0b", radius: "0px" }} />
```

## The token model

Every theme is a plain object of tokens:

```ts
interface GuardTheme {
  accent: string;      // primary interactive color
  accentText: string;  // text on top of accent
  surface: string;     // dialog surface
  backdrop: string;    // full-screen backdrop behind the dialog
  surfaceBlur: string; // backdrop-filter blur on the surface, e.g. "16px"
  text: string;
  textMuted: string;
  border: string;
  radius: string;      // dialog + control corner radius
  spacing: string;     // base spacing unit
  controlSize: string; // min size of interactive controls (keep ≥ 44px)
  fontFamily: string;
  fontSize: string;
}
```

Pass a preset name, a preset plus `themeOverrides`, or a complete `GuardTheme` object instead of a preset. The helpers are exported for building your own:

```ts
import {
  guardThemePresets,   // Record<GuardThemePreset, GuardTheme>
  resolveGuardTheme,   // (theme?, overrides?) => GuardTheme
  themeToCssVariables, // (theme) => Record<"--fhg-*", string>
  DEFAULT_THEME_PRESET, // "light"
} from "form-human-guard/react";

const custom = resolveGuardTheme("midnight", { accent: "#f472b6" });
```

## ShadowRoot notes

The overlay renders inside a **closed ShadowRoot**, so normal page CSS cannot reach it — that's the point. Theming works anyway because tokens are applied as `--fhg-*` CSS custom properties **inline on the overlay container, inside the shadow boundary**. Consequences:

- Your global stylesheet cannot restyle the overlay; use the theme tokens instead.
- The theme object is the entire styling surface — there are no escape hatches, which keeps the overlay's accessibility guarantees (focus trap, contrast, touch targets) intact.
- If you need a fully custom UI rather than a themed dialog, use the headless [`useSubmitGuard`](/docs/react#usesubmitguardoptions) hook and render your own markup in normal DOM.
